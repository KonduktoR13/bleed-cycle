Техническое задание: PWA-приложение для визуального календаря эмоционального состояния по менструальному циклу

Цель проекта

Нужно создать PWA-приложение, которое помогает мужчине лучше понимать возможные изменения эмоционального состояния жены в зависимости от фазы менструального цикла.

Приложение не должно ставить диагнозы, заменять врача или утверждать, что женщина “обязательно” будет чувствовать себя определённым образом в конкретный день. Оно должно показывать вероятностную, ориентировочную картину на основе закономерностей, описанных в файле "bleed_study.md".

Основной сценарий: пользователь вводит дату начала менструации. По желанию можно указать дату окончания менструации. На основе этих данных приложение строит календарь цикла с визуальным прогнозом эмоционального фона, уровня чувствительности, раздражительности, усталости, тревожности, социальной открытости, либидо и других расчётных характеристик.

Исходные данные

В проекте должен использоваться файл "bleed_study.md" как основной аналитический источник. Из него нужно извлечь прикладные закономерности и превратить их в расчётную модель для календаря.

Модель должна учитывать, что цикл индивидуален, а прогноз является вероятностным. Нельзя подавать расчёты как точное предсказание поведения конкретного человека.

Общая концепция приложения

Приложение должно быть максимально визуальным. Главная ценность — не таблицы и длинные тексты, а быстрое восприятие состояния по цветам, градиентам, календарю, карточкам и понятной легенде.

Главный экран должен выглядеть как современный календарь цикла. Каждый день должен иметь визуальную окраску или градиент, отражающий расчётный эмоциональный фон и/или уровень эмоциональной нагрузки.

Пользователь должен иметь возможность нажать на любой день календаря и увидеть подробную карточку этого дня: фаза цикла, ожидаемые эмоциональные особенности, возможные риски, рекомендации по общению и краткое объяснение расчёта.

Интерфейс должен быть современным, аккуратным, мобильным, удобным для Android-устройства. Визуальный стиль — спокойный, приватный, не медицински пугающий, но достаточно информативный.

Платформа и режим работы

Приложение должно быть PWA.

Оно должно устанавливаться на Android-устройство как приложение через браузер.

После установки приложение должно полноценно работать оффлайн.

Все данные пользователя должны сохраняться локально на устройстве. Никакие данные о цикле, жене, эмоциональных состояниях или истории не должны отправляться на внешний сервер.

Сервер нужен только временно для первичной установки приложения на Android-устройство.

Сервер должен запускаться локально на компьютере разработчика и быть доступен только в локальной Wi-Fi сети по адресу вида "192.168.x.x".

Порт сервер должен выбирать автоматически из свободных портов.

После установки PWA на Android сервер больше не должен быть нужен для обычного использования приложения.

Локальное хранение данных

Приложение должно хранить данные локально на Android-устройстве.

Нужно сохранять:

- даты начала менструаций;
- опциональные даты окончания менструаций;
- рассчитанную или заданную среднюю длину цикла;
- историю циклов;
- пользовательские заметки по дням;
- ручные корректировки состояния, если они будут добавлены;
- настройки визуализации;
- настройки приватности.

Данные должны сохраняться устойчиво: после закрытия приложения, перезагрузки устройства и работы без интернета они должны оставаться доступными.

Желательно предусмотреть экспорт и импорт локальных данных, например в JSON-файл, чтобы можно было перенести данные или сделать резервную копию.

Основной пользовательский сценарий

При первом запуске приложение должно предложить ввести дату начала последней менструации.

Дополнительно можно указать дату окончания менструации, но это не должно быть обязательным.

Если известна обычная длина цикла, пользователь может указать её вручную. Если нет — приложение использует стандартное значение по умолчанию, например 28 дней, но явно показывает, что это только стартовая оценка.

После ввода данных приложение строит календарь текущего цикла и ближайшего прогноза.

Пользователь видит календарь, где каждый день визуально окрашен по расчётному эмоциональному состоянию.

При нажатии на день открывается подробная карточка с характеристиками этого дня.

Когда начинается новая менструация, пользователь добавляет новую дату начала. Приложение обновляет среднюю длину цикла и постепенно делает прогноз более индивидуальным.

Календарь

Календарь является центральным экраном приложения.

Он должен показывать дни месяца или цикла в удобном мобильном формате.

Каждый день должен иметь визуальный индикатор состояния. Это может быть цвет, градиент, насыщенность, рамка, маленькие маркеры или комбинация этих элементов.

Нужно избегать примитивной логики “хороший день / плохой день”. Лучше показать несколько измерений: эмоциональная стабильность, чувствительность, энергия, раздражительность, тревожность, социальность, потребность в поддержке.

Календарь должен иметь легенду. Пользователь должен быстро понимать, что означают цвета и градиенты.

Пример логики визуального отображения:

- спокойные нейтральные дни — мягкие, сбалансированные цвета;
- дни повышенной энергии и социальной открытости — более светлые и тёплые оттенки;
- дни повышенной чувствительности — более насыщенный или мягко-контрастный оттенок;
- дни вероятного ПМС — более выраженный предупреждающий градиент;
- дни менструации — отдельный понятный визуальный слой;
- овуляторное окно — отдельная маркировка, не агрессивная и не чрезмерно акцентированная.

Важно: цвета не должны стигматизировать женщину. Нельзя делать интерфейс в духе “опасный день”. Формулировки должны быть уважительными и нейтральными.

Карточка дня

При нажатии на день должна открываться подробная карточка.

В карточке нужно показывать:

- дату;
- день цикла;
- фазу цикла;
- вероятный гормональный контекст;
- общий эмоциональный фон;
- уровень энергии;
- эмоциональную чувствительность;
- вероятность раздражительности;
- вероятность тревожности;
- социальную открытость;
- возможную потребность в отдыхе;
- возможную потребность в поддержке;
- рекомендации по общению;
- краткое объяснение, почему приложение показывает именно такой прогноз.

Формулировки должны быть мягкими. Например:

“Сегодня может быть выше чувствительность к критике.”

“Лучше избегать давления и резких формулировок.”

“Поддержка, спокойный тон и практическая помощь могут восприниматься особенно хорошо.”

“Прогноз ориентировочный и может не совпадать с реальным состоянием.”

Расчётные характеристики

Приложение должно рассчитывать несколько характеристик по каждому дню цикла.

Минимальный набор:

- эмоциональная стабильность;
- чувствительность;
- раздражительность;
- тревожность;
- уровень энергии;
- усталость;
- социальная открытость;
- либидо;
- потребность в поддержке;
- риск конфликтности;
- комфортность для серьёзных разговоров;
- комфортность для активных планов;
- вероятность ПМС-симптомов.

Каждая характеристика может рассчитываться по шкале, например от 0 до 100.

Важно, чтобы расчёт был не бинарным, а плавным. Между фазами цикла должны быть переходы, а не резкие скачки.

Фазы цикла

Модель должна использовать фазы цикла:

1. Менструальная фаза.
2. Ранняя фолликулярная фаза.
3. Поздняя фолликулярная фаза.
4. Овуляторное окно.
5. Ранняя лютеиновая фаза.
6. Средняя лютеиновая фаза.
7. Поздняя лютеиновая фаза / предменструальный период.

Для каждой фазы нужно определить типичные эмоциональные и поведенческие тенденции на основе "bleed_study.md".

Нужно учитывать, что фактическая овуляция не всегда происходит строго в середине цикла. Если точных данных об овуляции нет, приложение должно делать приблизительную оценку. Например, овуляция ориентировочно происходит примерно за 14 дней до следующей менструации, а не обязательно на 14-й день цикла.

Менструальная фаза

В дни менструации приложение может показывать повышенную вероятность физического дискомфорта, усталости, сниженной энергии, потребности в отдыхе и более бережном общении.

Эмоциональная картина может быть разной: у части женщин после начала менструации напряжение ПМС снижается, но физический дискомфорт может сохраняться.

Приложение должно различать:

- позднюю лютеиновую фазу перед началом менструации;
- собственно первые дни кровотечения;
- последние дни менструации.

Если пользователь указывает дату окончания менструации, приложение должно точнее отображать эту фазу.

Фолликулярная фаза

После завершения менструации приложение может показывать постепенное повышение энергии, улучшение настроения, увеличение мотивации, большей открытости к планам, общению и активности.

Визуально это может быть фаза “восстановления и роста энергии”.

Эта фаза обычно должна выглядеть в календаре более стабильной и позитивной, но без чрезмерных утверждений.

Овуляторное окно

В овуляторный период приложение может показывать повышенную социальную открытость, уверенность, либидо, энергию и общительность.

Также нужно учитывать, что у части женщин в это время могут быть овуляторные боли, чувствительность или перепады настроения.

Овуляторное окно не должно отображаться грубо или вульгарно. Это должна быть нейтральная физиологическая информация.

Лютеиновая фаза

В ранней и средней лютеиновой фазе состояние может быть относительно стабильным, но ближе к концу лютеиновой фазы может возрастать вероятность ПМС-симптомов.

В поздней лютеиновой фазе приложение должно показывать возможное увеличение:

- эмоциональной чувствительности;
- раздражительности;
- тревожности;
- усталости;
- сниженной стрессоустойчивости;
- потребности в поддержке;
- дискомфорта от критики;
- риска конфликтов;
- желания уединения или снижения социальной нагрузки.

Прогноз должен быть особенно осторожным в формулировках. Нельзя писать так, будто женщина “станет злой” или “будет неадекватной”. Нужно использовать вероятностные и уважительные формулировки.

ПМС и ПМДР

Приложение должно различать обычные предменструальные изменения, ПМС и тяжёлые состояния вроде ПМДР.

Нельзя диагностировать ПМС или ПМДР автоматически.

Можно добавить предупреждающий информационный блок: если эмоциональные симптомы каждый цикл выражены резко, мешают жизни, работе, отношениям, сопровождаются сильной депрессией, вспышками гнева, паникой или мыслями о самоповреждении, это повод обратиться к врачу.

Этот блок должен быть оформлен аккуратно, без запугивания.

Индивидуализация

Приложение должно начинать с общей модели, но постепенно становиться более персональным.

Даже если на первом этапе пользователь вводит только дату начала менструации, архитектура продукта должна позволять в будущем добавлять ручную оценку реального состояния по дням.

Желательно предусмотреть возможность, чтобы пользователь отмечал:

- реальное настроение;
- уровень энергии;
- раздражительность;
- тревожность;
- боль;
- качество сна;
- стресс;
- заметки.

На основе этих данных в будущем можно будет корректировать прогноз под конкретную женщину.

На первом этапе это может быть необязательной функцией, но её стоит предусмотреть в структуре приложения.

Визуализация

Визуализация должна быть главным преимуществом приложения.

Нужно использовать:

- календарь с цветовым кодированием;
- плавные градиенты;
- легенду;
- карточки дня;
- шкалы характеристик;
- компактные графики;
- фазовую линию цикла;
- визуальное различение фактических и прогнозируемых дней.

Хорошая идея: сделать общий “эмоциональный индекс дня”, но не ограничиваться только им. Индекс должен быть входом в цвет календаря, а подробные характеристики должны быть доступны в карточке дня.

Цветовая система должна быть понятной:

- не использовать агрессивный красный как “женщина опасна”;
- не делать стигматизирующих обозначений;
- использовать мягкие, современные, читаемые оттенки;
- предусмотреть понятную легенду;
- обеспечить достаточный контраст для читаемости.

Рекомендации по общению

Для каждого дня приложение может показывать краткие рекомендации мужчине.

Рекомендации должны быть практичными, уважительными и не манипулятивными.

Примеры типов рекомендаций:

- лучше говорить спокойнее;
- не начинать тяжёлый разговор без необходимости;
- больше помогать практически;
- не обесценивать усталость;
- избегать давления;
- быть внимательнее к тону;
- предложить отдых;
- не воспринимать резкость автоматически как личную атаку;
- выбирать более мягкий момент для сложных тем;
- поддержать, но не навязываться.

Важно: приложение не должно учить “управлять женщиной”. Оно должно помогать лучше понимать возможное состояние и выбирать более бережное поведение.

Приватность

Так как данные чувствительные, приложение должно выглядеть и работать приватно.

Желательно предусмотреть:

- локальное хранение;
- отсутствие внешней аналитики;
- отсутствие отправки данных в интернет;
- возможность очистить все данные;
- возможность экспортировать данные;
- возможность импортировать данные;
- простой экран “Приватность”, где объясняется, что данные остаются на устройстве.

Можно добавить возможность включить PIN-код или простую блокировку приложения, если это технически уместно для PWA.

Оффлайн-режим

PWA должно уверенно работать без интернета.

После установки должны быть доступны:

- главный экран;
- календарь;
- история циклов;
- расчёты;
- карточки дней;
- настройки;
- локальные данные;
- легенда;
- справочная информация.

При отсутствии сети приложение не должно ломаться или показывать пустой экран.

Установка через локальный сервер

Нужно реализовать удобный локальный запуск сервера для установки PWA.

Требования:

- сервер запускается локально;
- доступен в Wi-Fi сети по адресу вида "http://192.168.x.x:порт";
- порт выбирается автоматически из свободных;
- после запуска в консоли должно быть ясно показано, по какому адресу открыть приложение на Android;
- сервер не должен быть рассчитан на публичный интернет;
- сервер нужен только для установки и обновления PWA;
- после установки приложение должно работать автономно.

Интерфейс

Интерфейс должен быть на русском языке.

Стиль: современный, минималистичный, визуальный, мобильный.

Главные экраны:

1. Первый запуск / ввод данных.
2. Главный календарь.
3. Карточка выбранного дня.
4. История циклов.
5. Настройки.
6. Справка / легенда.
7. Приватность / данные.

Первый запуск должен быть очень простым. Пользователь не должен заполнять длинную анкету.

Минимальный ввод:

- дата начала последней менструации.

Опционально:

- дата окончания;
- обычная длина цикла;
- заметка.

Главный экран

На главном экране желательно показать:

- текущий день;
- день цикла;
- текущую фазу;
- общий эмоциональный индекс;
- календарь;
- легенду или кнопку легенды;
- ближайшие дни с изменениями;
- кнопку добавления новой менструации.

Главный экран должен за несколько секунд давать понимание: какая сейчас фаза, какой эмоциональный фон вероятен, насколько день чувствительный и как лучше общаться.

История циклов

Экран истории должен показывать прошлые циклы.

Для каждого цикла:

- дата начала;
- дата окончания, если известна;
- длина цикла;
- длительность менструации;
- отклонение от среднего;
- заметки.

Приложение должно рассчитывать среднюю длину цикла по истории, если есть достаточно данных.

Если данных мало, нужно ясно указывать, что прогноз приблизительный.

Настройки

В настройках должны быть:

- средняя длина цикла;
- длительность менструации по умолчанию;
- уровень детализации подсказок;
- включение/отключение отдельных характеристик;
- визуальная тема;
- экспорт данных;
- импорт данных;
- очистка данных;
- информация о приватности.

Тон текста в приложении

Тон должен быть уважительным, спокойным и взрослым.

Нельзя использовать грубые или стереотипные формулировки.

Не использовать формулировки вроде:

- “опасный день”;
- “она будет злая”;
- “лучше не подходить”;
- “женская истерика”;
- “гормоны управляют женщиной”.

Лучше использовать:

- “возможна повышенная чувствительность”;
- “стресс может переноситься тяжелее”;
- “лучше выбирать мягкий тон”;
- “может быть выше потребность в отдыхе”;
- “прогноз ориентировочный”.

Ограничения и дисклеймер

В приложении должен быть короткий дисклеймер:

“Это приложение показывает ориентировочный прогноз на основе фаз менструального цикла и общих закономерностей. Оно не является медицинским инструментом, не ставит диагнозы и не заменяет консультацию врача. Реальное состояние человека зависит от сна, стресса, здоровья, отношений, работы, питания и других факторов.”

Дисклеймер не должен мешать использованию, но должен быть доступен.

Качество и ожидаемый результат

В результате должен получиться законченный PWA-прототип, который можно открыть с Android-устройства через локальный Wi-Fi, установить на устройство и затем использовать оффлайн.

Приложение должно выглядеть как современный мобильный продукт, а не как техническая демо-страница.

Ключевые критерии результата:

- работает оффлайн после установки;
- данные сохраняются локально;
- сервер нужен только для установки;
- календарь визуально информативный;
- есть градиенты, цвета и легенда;
- можно нажимать на дни;
- по каждому дню есть подробные расчётные характеристики;
- используется логика из "bleed_study.md";
- интерфейс русский;
- формулировки уважительные и не стигматизирующие;
- есть история циклов;
- есть настройки;
- есть экспорт/импорт данных;
- приложение удобно использовать на Android.

Приоритеты

Главный приоритет — визуальное понимание информации через календарь, цвета, градиенты и карточки дней.

Второй приоритет — надёжная оффлайн-работа и локальное сохранение данных.

Третий приоритет — корректная, осторожная и уважительная интерпретация эмоциональных закономерностей.

Код, архитектура и техническая реализация остаются на усмотрение Codex, но итоговое приложение должно соответствовать описанному пользовательскому опыту.
